База знаний

01. Основы Linux терминала и навигации

Командная строка — это главный инструмент любого исследователя безопасности. В отличие от графического интерфейса, терминал позволяет автоматизировать рутину и управлять системой напрямую.

Основные команды навигации:

pwd

Показывает текущую директорию, в которой вы находитесь.

ls -la

Выводит список всех файлов и папок в директории, включая скрытые (начинающиеся с точки), а также показывает права доступа.

cd /path/to/folder

Переход в указанную папку. Использование команды cd .. поднимет вас на один уровень вверх.

Управление правами доступа (Важно для запуска скриптов):

Если загруженный из сети скрипт не запускается, ему нужно выдать права на исполнение:

chmod +x script.py
02. Основы анализа сети и сканирования

Правильный анализ сетевой инфраструктуры позволяет обнаружить активные хосты и выяснить, какие порты на них открыты, для дальнейшего анализа безопасности.

Определение сетевых интерфейсов:

Прежде чем сканировать сеть, нужно узнать собственный IP-адрес и имя сетевой карты:

ip a

Сканирование с помощью утилит (концепт):

Обычно в тестах на проникновение используется сканирование портов для обнаружения версий запущенных служб. Пример классического синтаксиса для сканирования сети:

nmap -sV -O 192.168.1.1

Где ключи означают:

  • -sV — запуск определения версий запущенных служб.
  • -O — попытка определить операционную систему целевого узла.
03. Аудит стойкости аутентификации

Данный материал демонстрирует важность использования сложных паролей и регулярной смены дефолтных учетных данных в сетевых устройствах.

Проверка криптографических хэшей:

В процессе тестирования систем часто проверяется скорость перебора различных типов хэшей (MD5, SHA-256, bcrypt) на локальных мощностях (CPU/GPU) для оценки времени, которое может потребоваться злоумышленнику в случае утечки базы данных.

Защита систем:

Для предотвращения подбора паролей на живых сервисах применяются жесткие политики:

  • Блокировка IP-адреса после 3-5 неудачных попыток входа (например, через Fail2ban).
  • Обязательное внедрение двухфакторной аутентификации (2FA).
  • Длина пароля администратора не менее 12-16 символов, включая разные регистры, цифры и спецсимволы.
